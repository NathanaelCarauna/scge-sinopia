1.​

Introdução
Este projeto visa desenvolver um Sistema Integrado de Ouvidoria Pública

baseado em Inteligência Artificial para cidadãos e servidores públicos que ofereça
eficiência e qualidade, ao mesmo tempo em que busca produzir os subsídios
institucionais. O sistema pretende endereçar demandas reprimidas relacionadas à
volume, coerência e resolutividade, além de explorar plenamente os conhecimentos
armazenados nas bases de dados da SGCE, ainda subutilizadas. Para os cidadãos
será possível automatizar, personalizar, melhorar o volume e a qualidade dos seus
pedidos e/ou manifestações. Em relação aos servidores públicos, será possível
gerar projeções de demandas que possam ser utilizadas para a gestão de recursos,
analisar coerência, promover auditabilidade, avaliação de indicadores chave para
aprimorar a eficiência operacional; e fornecer insights para a sustentabilidade e
melhoria contínua dos serviços públicos e da gestão governamental.
1.1.​

Escopo e Alinhamento Estratégico
O Sistema Integrado de Ouvidoria Pública é um projeto robusto que

será dividido em quatro módulos para garantir a melhor estruturação e
eficiência do projeto, sendo eles o OE1: Atendimento Multimodal; OE2:
Suporte ao Ouvidor; OE3: Suporte, Controle e Melhoria Operacional e o OE4:
Apoio ao Gestor. Esse documento de requisito irá focar nos requisitos do
módulo OE1: Atendimento Multimodal.
O sistema permitirá:
●​ Gerenciamento de usuários por perfil (Usuário não cadastrado, Usuário
não identificado, Logado, Logado Gov, Logado Ouve);
●​ Controle de acesso baseado no tipo de autenticação feita pelo cidadão;
●​ Receber,

analisar

e

encaminhar

pedidos

e/ou

manifestações

multimodais (i.e., voz e texto) dos cidadãos;
●​ Acompanhamento dos pedidos e/ou manifestações realizadas;

2. Visão Geral do Produto
2.1.​

Perspectivas do produto

O sistema será uma aplicação web/mobile integrado de ouvidoria pública,
acessível por navegadores modernos, destinada a atender cidadãos e
servidores públicos. O sistema terá funcionalidades distintas por perfil de
usuário, incluindo análise, encaminhamento e acompanhamento de pedidos
e/ou manifestações multimodais dos cidadãos.

4. Requisitos Funcionais (Casos de Uso)
Integração com OUVE.PE

7

[RF01] - Conexão com API OUVE.PE
●​ Descrição
O plugin deve se conectar com as APIs existentes do sistema
OUVE.PE.
[RF02] - Sincronização de dados
●​ Descrição
O plugin deve sincronizar dados bidirecionalmente com OUVE.PE.
[RF03] - Autenticação integrada
●​ Descrição
O plugin deve utilizar o sistema de autenticação existente do
OUVE.PE.
[RF04] - Mapeamento de usuários
●​ Descrição
O plugin deve mapear usuários entre os sistemas sem duplicação.

Registro e Gestão de Manifestações
[RF05] - Classificação assistida
●​ Descrição
O plugin deve sugerir classificações de manifestações baseado no
conteúdo dentro das 5 categorias: Reclamação, Solicitação, Denúncia,
Sugestão e Elogio. Caso não seja a desejada, ter a opção de alterar.
[RF06] - Manifestação por voz
●​ Descrição
O plugin deve permitir registro completo de manifestações por
comando de voz.

8

[RF07] - Assistente de preenchimento
●​ Descrição
O plugin deve assistir no preenchimento da manifestação, dando
sugestões inteligentes e validação em tempo real.
[RF08] - Triagem automática da manifestação
●​ Descrição
O sistema deve ser capaz de efetuar a triagem do órgão competente
de acordo com o conteúdo da manifestação.
[RF09] - Direcionamento ao portal da transparência
●​ Descrição
O plugin deve ser capaz de identificar quando uma informação
solicitada já se encontra no portal da transparência e direcionar o
usuário para o portal correspondente.
[RF10] - Alerta sobre finalização de processo
●​ Descrição
O plugin deve ser capaz de emitir um alerta ao usuário para que ao
final do processo de manifestação, ele não perca informações
importantes ao fechar.
[RF11] - Cópia de informações do processo
●​ Descrição
O plugin deve ser capaz de baixar uma cópia na máquina do usuário
com as informações da manifestação, como protocolo e senha.

Avatar Inteligente e Interface Multimodal
[RF12] - Avatar de atendimento
●​ Descrição

9

O plugin deve disponibilizar avatar inteligente sobreposto à interface
OUVE.PE.
[RF13] - Atendimento por voz
●​ Descrição
O plugin deve capturar comandos de voz e convertê-los em ações no
OUVE.PE.
[RF14] - Chatbot integrado
●​ Descrição
O plugin deve oferecer chatbot que compreende o contexto do
OUVE.PE.
[RF15] - Síntese de fala
●​ Descrição
O plugin deve converter texto da interface em áudio para usuários com
deficiência visual.
[RF16] - Controle do Plugin
●​ Descrição
O plugin deve ser capaz de ser desativado e reativado pelo usuário.
[RF17] - Guia da Plataforma
●​ Descrição
O plugin deve oferecer um guia do usuário em como se movimentar na
plataforma e o que significa cada módulo.

Acessibilidade e Inclusão Digital
[RF18] - Leitor de tela
●​ Descrição

10

O plugin deve enriquecer informações para leitores de tela, fazendo
descrições contextuais.
[RF19] - Alto contraste
●​ Descrição
O plugin deve oferecer alternância de tema com alto contraste.
[RF20] - Ajuste de fonte
●​ Descrição
O plugin deve oferecer ajuste de fonte para aumentar a letra dos textos

O plugin deve oferecer interface simplificada para neuro divergentes,

[RF23] - Feedback sonoro
●​ Descrição
O plugin deve fornecer feedback auditivo para ações no sistema, como

[RF24] - Zoom e magnificação
●​ Descrição
O plugin deve permitir zoom avançado de elementos da interface.

Pesquisa e Análise
[RF25] - Busca avançada
●​ Descrição
O sistema deve permitir a busca avançada de manifestações e
atendimentos com filtros por data e status.
[RF26] - Análise de veracidade
●​ Descrição
O sistema deve analisar o sentimento das manifestações para verificar
a

veracidade

das

informações

oferecidas,

classificando

uma

ambos os casos.
[RF27] - Detecção de duplicatas
●​ Descrição
O sistema deve identificar manifestações com o mesmo conteúdo já

●​ Descrição

●​ Descrição
